---
command: make-report
title: Make a report
description: Summarize what is here, what matters, what looks risky, and what to do next.
hidden: true
---

Create a concise operator-facing report for the current workspace.

Include:
- What is present and how the pieces relate.
- The datasets, signals, schemas, or pipelines that matter most.
- Risks, anomalies, missing context, or implementation issues worth addressing.
- Recommended next actions, ordered by practical impact.

Keep the report specific to observed files and command outputs. Separate confirmed facts from hypotheses.
